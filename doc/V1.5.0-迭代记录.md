## 一、迭代背景

原小程序后端为独立 Spring Boot 2.7.18 应用（端口 4001），现已整合到 ruobing-boot 多模块框架中作为 codemanual 子模块（端口 8080）。本次迭代将小程序从旧后端迁移到新后端。

## 二、架构变化

### 2.1 后端整合前后对比

| 维度 | 旧后端（独立） | 新后端（ruobing-boot） |
|------|--------------|----------------------|
| 端口 | 4001 | 8080 |
| 上下文路径 | `/api` | `/codemanual` |
| 完整地址 | `http://localhost:4001/api` | `http://localhost:8080/codemanual` |
| 认证方式 | openid 参数传递 | Token（Authorization Header） |
| 成功响应码 | `code: 200` | `code: 0` |
| 用户表 | `user` | `sys_rb_manual_user` |

### 2.2 认证方案对比

**旧方案（独立后端）**：
```
小程序 ──code──→ 后端 /user/login ──openid──→ 后续请求带 openid 参数
```

**新方案（ruobing-boot）**：
```
小程序 ──code──→ 后端 /auth/wx-login ──accessToken──→ Redis ──后续请求 Authorization: token
```

## 五、Token 管理机制

### 5.1 存储
```javascript
// 登录成功后
wx.setStorageSync('accessToken', data.accessToken)
wx.setStorageSync('userInfo', userInfo)
```

### 5.2 使用
```javascript
// request.js 自动注入
const token = wx.getStorageSync('accessToken') || ''
header: {
  'Content-Type': 'application/json',
  'Authorization': token  // 自动携带
}
```

### 5.3 失效处理
```javascript
// request.js 中检测到 code:401 自动清除
if (res.data && res.data.code === 401) {
  app.clearAuth && app.clearAuth()
}
```

## 六、响应码兼容策略

新旧后端成功响应码不同，统一兼容处理：

```javascript
function isApiSuccess(res) {
  if (!res || res.code === undefined) return false
  const c = res.code
  return c === 0 || c === 200 || c === '0' || c === '200'
}
```

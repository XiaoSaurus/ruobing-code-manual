## 🐛 Bug 修复

### 1. 视图切换不生效（全局样式优先级冲突）

**问题描述**：小程序列表页的"单列/双列"切换按钮点击后，页面仍保持单列排列，双列样式完全不生效。

**根本原因**：`app.wxss` 中定义了全局 `.card` 样式，其优先级与页面局部样式相同但先加载，覆盖了页面中定义的双列 `display: flex` 和 `width: 48%`。此外，`box-sizing` 未显式设置，导致 `padding` 被叠加到 `width` 之外。

**修复方案**：
- 所有视图切换相关选择器添加 `!important` 强制优先级
- 所有卡片元素添加 `box-sizing: border-box`
- 修复文件：`web-design/list.wxss`、`graduation/list.wxss`

```css
/* 修复后的双列容器 */
.card-list.view-double {
  display: flex !important;
  flex-direction: row !important;
  flex-wrap: wrap !important;
  justify-content: space-between !important;
  box-sizing: border-box;
}

/* 修复后的双列卡片 */
.card-list.view-double .card-double {
  width: 48%;
  box-sizing: border-box;
}
```

### 2. 响应格式不一致

**问题描述**：`web-design` 和 `graduation` 两个模块的接口返回格式完全不同，导致首页列表加载失败。

**差异**：
- `/web-design/hot` 返回：数组 `[{ ... }]`
- `/graduation/hot` 返回：分页对象 `{ records: [...], total: N }`

**修复方案**：在小程序端统一使用 `normalize()` 函数处理响应：

```javascript
function normalize(res) {
  if (res.code !== 200) return Promise.reject(res);
  const data = res.data || [];
  // graduation 接口的 data 是分页对象，取 records
  return Array.isArray(data) ? data : (data.records || []);
}
```

## ✨ 新增功能

### 1. 列表页视图切换（单列 / 双列瀑布流）

- 点击切换按钮在单列和双列瀑布流之间切换
- 单列：大卡片，封面图高度 320rpx，适合浏览详情
- 双列：双列瀑布流，封面图高度 200rpx，适合快速扫视
- 状态通过 `viewMode`（`'single'` / `'double'`）本地保存

**修改文件**：
- `web-design/list.js` — 添加 `toggleView()`、初始化 `viewMode`
- `web-design/list.wxml` — 切换按钮 UI
- `graduation/list.js` — 同上
- `graduation/list.wxml` — 同上
- `web-design/list.wxss` — 双列瀑布流样式
- `graduation/list.wxss` — 同上

### 2. 下拉自动加载分页

- 移除传统的"加载更多"按钮
- 改用 `onReachBottom` 事件，滑动到底部自动加载下一页
- 底部显示三个跳动圆点 + "正在加载..." 动画
- 去掉全屏 `wx.showLoading`，用户可边浏览边加载
- 每页 10 条数据

### 3. 搜索栏 Apple 风格改造

- 背景色：`#f5f5f7`（淡灰）
- 无边框，图标淡灰色
- 圆角统一 12rpx
- placeholder 文案精简

# 若冰代码手册小程序：迭代问答流程（V1.5.0 起）

## 1. 适用范围
- 本流程用于本项目后续每次「提出需求 / 回答方案 / 执行改动 / 验证结果」。
- 所有改动默认基于微信小程序工程 `mini-program`。

## 2. 标准流程

### 步骤 A：需求确认（问）
- 明确版本号（例如：`V1.5.0`）。
- 明确页面或模块（例如：`pages/mine/login`）。
- 明确改动目标（做什么）与禁改范围（不做什么）。
- 若需求有歧义，先补充 1-2 个关键问题再执行。

### 步骤 B：影响分析（答）
- 先定位涉及文件：`wxml / js / wxss / json`。
- 给出改动清单：删除项、保留项、新增项。
- 说明潜在联动影响（接口、状态字段、样式、文案）。

### 步骤 C：实施修改（做）
- 先改结构（WXML），再改逻辑（JS），最后改样式（WXSS）。
- 清理无用状态与方法，避免保留僵尸代码。
- 文案调整与交互保持一致，避免“页面已删但提示还在”。

### 步骤 D：自检验收（验）
- 检查编译是否通过、页面是否可打开。
- 验证主路径：登录按钮、协议勾选、异常提示。
- 确认已删除项不再出现（UI 与逻辑都要确认）。

### 步骤 E：结果输出（回）
- 输出内容必须包含：
  - 修改了哪些文件；
  - 每个文件的关键改动点；
  - 验证结果与未覆盖风险（如有）。

## 3. 回答模板（后续默认使用）
